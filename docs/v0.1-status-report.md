## Residual limitations (not blockers)

Documented in [`maestro-guide.md`](maestro-guide.md#limitações-conhecidas-e-drift-documental):

1. Design doc recommends Go; v0.1 is Python.
2. Detached supervisor mechanism differs from spec wording (double-fork vs `_supervise`).
3. PID reuse: liveness probe only.
4. `rm --force` does not delete logs (spec MAY).
5. Detached `run` stdout differs from spec §3.8 worked example.
6. Repo/path name `mastro` vs product name Maestro.
7. Editable pip install needs recent pip/setuptools.

Spec Part 4 done-criteria checkboxes remain unchecked in the spec file itself;
functionally the criteria are met per tests and closed issues. A future pass may
tick them or replace with a release tag.
